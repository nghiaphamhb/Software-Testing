# Mockito: что ещё важно помнить

## Основные методы
- `mock()` — создать фейковый объект
- `spy()` — обернуть реальный объект
- `when(...).thenReturn(...)` — задать результат
- `doReturn(...).when(...)` — безопаснее для `spy`
- `verify(...)` — проверить вызов
- `times(n)` — сколько раз вызвали
- `never()` — не должен вызываться
- `any()` — любой аргумент
- `eq(x)` — точный аргумент `x`
- `isNull()` / `notNull()` — проверка `null`

## Полезные проверки
- `verify(service).save()` — метод вызван
- `verify(service, times(2)).save()` — вызван 2 раза
- `verify(service, never()).delete()` — не вызывался
- `verifyNoMoreInteractions(mock)` — лишних вызовов нет
- `verifyNoInteractions(mock)` — вообще не трогали

## Для исключений
- `thenThrow(...)` — выбросить исключение
- `doThrow(...).when(...)` — для `void`-методов

## Для void-методов
- `doNothing().when(...)`
- `doThrow().when(...)`
- `doAnswer().when(...)`

## Захват аргументов
- `ArgumentCaptor` — поймать аргумент и проверить его потом

Пример:
- проверить не только вызов,
- но и **что именно передали**.

## Что легко забыть
- Если используешь `any()`, тест может стать слишком слабым
- Если используешь matcher (`any`, `eq`) — используй matcher для **всех** аргументов
- Для `spy` лучше `doReturn(...).when(...)`, а не `when(...).thenReturn(...)`

## Как быстро запомнить
- **mock** = всё фейковое
- **spy** = объект реальный, но под наблюдением
- **when** = что вернуть
- **verify** = что было вызвано
- **captor** = что реально передали
